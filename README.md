# Tab Audio Recorder

Chrome / Edge 浏览器扩展——一键录制当前标签页音频，保存为本地 `.webm` (Opus) 文件。

适合在浏览器中观看直播、网课、知识视频时，一边正常收听，一边把当前标签页的声音录下来。录完后可手动上传到通义听悟、飞书妙记等转写服务。

## 功能特性

- **一键录制**：点击扩展图标 → 开始录制，操作极简
- **暂停 / 恢复**：录制中可随时暂停，暂停期间网页声音正常播放
- **手动导出**：停止录制后录音不会自动下载，确认需要后再导出
- **快捷键支持**：无需打开弹窗也能控制录音
- **不影响收听**：录音期间标签页声音正常播放，戴耳机也能听到
- **精准隔离**：只捕获目标标签页的音频流，不录其他标签页、不录系统声音、不录麦克风
- **音量无关**：在浏览器内部截取音频数据流，录音质量不受系统音量影响
- **纯本地**：不上传、不联网、零隐私风险

## 快捷键

| 快捷键 | 功能 |
|---|---|
| `Alt + Shift + 9` | 开始 / 停止录制 |
| `Alt + Shift + 8` | 暂停 / 恢复录制 |

快捷键在弹窗关闭时也能使用，操作结果会通过系统通知提示。

## 使用方法

1. 打开目标网页（视频、直播、音频等），确认页面正在播放且**标签页未被静音**。
2. 点击浏览器工具栏中的扩展图标，打开弹窗。
3. 点击「**开始录制当前标签页**」。
4. 录制过程中：
   - 可以点击「暂停录制」，暂停后点击「继续录制」。
   - 可以关闭弹窗，录音不会中断。
   - 工具栏图标会显示状态标记（REC / PAU / OK）。
5. 点击「**停止录制**」，此时录音文件暂存在内存中，不会立刻下载。
6. 点击「**导出录音文件**」，浏览器会下载一个 `.webm` 文件。
7. 文件名格式：`网页标题_YYYY-MM-DD_HH-mm-ss.webm`。

下载后的文件可用 Chrome 浏览器、VLC、PotPlayer 等播放器直接打开。

## 兼容性

### 操作系统

Windows、macOS、Linux 均可——扩展运行在浏览器内部，不依赖操作系统特性。

### 浏览器

| 浏览器 | 支持情况 |
|---|---|
| Chrome 116+ | 完全支持（主要开发目标） |
| Edge（Chromium 内核）116+ | 完全支持 |
| Brave / Vivaldi / Arc 等 Chromium 系 | 预期可用（未逐一验证） |
| Firefox | 不支持（API 体系不同） |
| Safari | 不支持（扩展机制不同） |

扩展依赖 `chrome.tabCapture` 和 `chrome.offscreen` 这两个 Chromium 专有 API，因此仅限 Chromium 内核浏览器使用。

## 工作原理

扩展使用 Chrome 的 `tabCapture` API 在浏览器内部截取指定标签页的音频数据流，然后通过 `MediaRecorder` 编码为 WebM/Opus 格式。

**这意味着：**

- 只录目标标签页的声音——其他标签页、其他软件的声音完全不会混入。
- 录音在音频源头截取，不经过系统音频层——系统音量大小不影响录音质量。
- 即使耳机音量开到最小，录出来的音量也是正常的。
- 唯一录不到的情况：标签页本身被静音（右键标签 → 将此标签页静音）。

## 技术细节

- **Manifest V3**：使用 Service Worker 而非 Background Page。
- **三层架构**：background（状态协调）→ offscreen（录音引擎）→ popup（展示层），通过 `chrome.runtime.sendMessage` 通信。
- **零依赖**：纯原生 JavaScript，无 npm、无构建步骤、无框架。
- **录音驻留内存**：完整录音的 blob 在导出前一直保存在内存中。长时间录音（数小时）会占用较多内存。

## 注意事项

- 录制前请确保标签页**未被静音**，否则无法捕获音频。
- DRM 保护内容（如部分 Netflix / Disney+ 内容）可能无法被捕获。
- 浏览器内置页面（`chrome://`、`edge://`）和扩展页面无法被录制。
- 长时间录音会持续占用浏览器内存，建议单次录制不超过 2 小时。
- 暂停录制只会暂停写入录音文件，不会暂停或静音网页播放。

## 隐私说明

- 扩展**不上传**任何数据，不发起任何网络请求。
- 扩展**不录制**麦克风或系统全局声音，仅捕获指定标签页的音频。
- 录音文件仅通过浏览器下载机制保存到本地。
- 扩展不收集、不存储、不传输任何个人信息。
