# PRD — v1-launch

## 概述

首版浏览器扩展，提供当前标签页音频的录制、暂停/恢复、停止和本地导出功能。

## 目标

- 用户能在浏览器内一键录制当前标签页音频
- 录音期间不影响正常收听
- 录音文件以 `.webm`（Opus 编码）格式保存到本地

## 功能清单

### F1 — 开始录制

- 点击 popup 中「开始录制当前标签页」按钮启动
- 快捷键 `Alt+Shift+9` 可在无 popup 时切换录制
- 自动获取当前标签页标题作为文件名前缀
- 状态指示器变红 + 工具栏 badge 显示「REC」

### F2 — 暂停 / 恢复

- 录制中可暂停，暂停期间网页声音仍正常播放
- 快捷键 `Alt+Shift+8` 切换暂停/恢复
- 暂停时状态指示器变橙 + badge 显示「PAU」
- 计时器只计算实际录制时长，跳过暂停段

### F3 — 停止录制

- 停止后生成待导出录音，不自动下载
- 如果录制目标丢失（标签页关闭、音频流中断），自动停止并提示用户导出
- 状态指示器变绿 + badge 显示「OK」

### F4 — 导出录音

- 用户确认后点击「导出录音文件」触发下载
- 文件名格式：`{页面标题}_{YYYY-MM-DD_HH-mm-ss}.webm`
- 主下载通道：`chrome.downloads` API
- 兜底通道：offscreen 内 `<a download>` 点击

### F5 — 重置

- 丢弃当前待导出录音，回到空闲状态
- 清理所有 objectUrl、关闭 offscreen 文档

### F6 — 信息展示

- popup 显示：当前标签页标题、输出格式、保存方式、文件状态
- 底部消息区显示操作提示和错误信息

## 限制与已知约束

- DRM 保护内容、浏览器内置页面、扩展页面无法被捕获
- 静音标签页无法录音（启动前检测并提示）
- 录音全程驻留内存，长时间录制会占用较多内存
- 需要 Chrome / Chromium-Edge 116+

## 验收标准

- [x] 能正常录制 YouTube / B站等主流视频网站的标签页音频
- [x] 录音期间用户能正常听到声音
- [x] 暂停/恢复后录音文件完整连续
- [x] 关闭 popup 不中断录音
- [x] 导出的 .webm 文件能用 Chrome / VLC 正常播放
- [x] 快捷键在无 popup 时能正常工作
